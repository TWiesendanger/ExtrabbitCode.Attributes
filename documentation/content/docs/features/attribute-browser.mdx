The Attribute Browser is the main panel of the add-in. It displays a live tree of all attributes attached to the currently active Inventor document, grouped by owner object and then by attribute set.

Open the panel from the **ExtrabbitCode.Inventor.Attributes** ribbon tab by clicking the **Attribute Dialog** button. The tree populates automatically when a document is active (depends on settings).

## Expand / Collapse All

The toolbar provides **Expand All** and **Collapse All** buttons that open or close every node in the tree at once.

{/* Screenshot: toolbar with Expand All and Collapse All buttons highlighted */}

| Action | Result |
|---|---|
| **Expand All** | Opens every document, owner, and attribute set node so all attributes are visible |
| **Collapse All** | Closes all nodes back to just the document root |

These are useful when you want a quick overview of everything in a large document, or when you want to tidy up the view after browsing.

<Callout type="warn">
  On documents with a large number of attributes, **Expand All** can take a noticeable amount of time. For large assemblies, consider using the [search filter](/docs/features/search-filter) to find what you need instead.
</Callout>

## Context menu

Right-clicking any node in the tree opens a context menu. The available items depend on the node type:

| Action | Available on | Description |
|---|---|---|
| **Select in Window** | All nodes | Selects the corresponding Inventor object in the Inventor viewport |
| **Edit** | Attribute nodes only | Opens the edit dialog pre-filled with the current value |
| **Delete** | Attribute and attribute set nodes | Deletes the attribute or the entire attribute set |

## Selecting items

Click any node to select it. The selection context determines which toolbar actions are available — for example, the **Delete** action is only enabled when an attribute or attribute set is selected.

## Refreshing the tree

The tree refreshes automatically when you switch to a different open document (if [Auto-Refresh](/docs/features/auto-refresh) is enabled). You can also trigger a manual refresh at any time using the refresh button in the toolbar.
